Closures, Yield, and the Enumerable Module

I tried to make a class Enumerable by dynamically mixing in the Enumerable module and defining #each after declaring the class. I failed. This is my story.

I was messing around with dynamically mixing in Ruby modules. Yep, I was pretty bored. Anyway, one of the most useful modules in Ruby is the Enumerable module. It’s a total badass. So I thought I’d try adding Enumerable to a class that I had already defined and see if that worked out.

The first step is including it dynamically. Maybe I can just call the include method on my class!

Okay! That appears to work. No exceptions. Now let’s use it. I’ll try out using #reduce, one of my favorite higher-order functions in Ruby.

Oops. Right. In order to take advantage of the Enumerable module, you need to include the module and also define an #each method.[^1] This makes a lot of sense - how can your class be enumerable in there’s nothing to enumerate?

Fine, let’s do that. I could just define it in my class definition and be done with it, but I’m having fun working with dynamically modifying classes, so I want to keep going with this.
